# Badger Research

Badger ReSearch is a website to connect with other researchers inside UW through connecting with their research interests. In the web-page, you can create a profile, and track the topics you are interested in. You can also find other UW researchers, and their preferred research tracks. Finally, a profile has a Whiteboard, an input system that allows users to create nodes of information and keep notes on their own research.

The project will consist of a main page with a login system and a create an account button. After logging in, the user gets taken to their main page, with their account information, their research topics, and their whiteboard.

I’m curious about how the fetching of the API will work, and how creating new accounts would work. My plan is to make some dummy accounts so that I can test out “connecting” to those accounts (a Search function inspired by the one in HW2).

The scope of this project might change depending on the difficulties found along the road. The bare minimum I plan on executing is a page where you can add your own research interests, and search for users with matching interests. The maximum scope would encompass a website that has an account creator, a research matcher, and an interactive whiteboard that can be used to annotate research ideas.
